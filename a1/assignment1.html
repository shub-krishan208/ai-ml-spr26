Problem-Statement:

A man has to cross a river using a boat. He carries three things with him, a sheep, a lion, a heap of grass. While crossing the river, he can take at most one thing with him on the boat at a time. Problem is, some of these objects are not to be left without survielance (let’s call these: incompatible).

We need to find an optimal way for the man to cross the river.

Mathematical Representation

The problem can be mathematically represented as states.

Let, any object on the Left bank has the state {1} and on the Right bank has state {0}. we have four objects, so initial state is {1,1,1,1} and goal state is {1,1,1,1}

        Left Bank           River           Right Bank
        state: 1 ------->  ~~~~~~ ------->  state: 0

         :initial:                            :Final:
        {1, 1, 1, 1}                        {0, 0, 0, 0}

So, we will have a graphical network of \(\space2^4\space\) nodes.

For simplicity and keeping the notations clean, let’s use an upper case alphabet as label for each node state.

Each node has the form: {x, x, x, x} => {man, sheep, grass, lion}

So, we have the Relation:

Constraints: We introduce constraints as defined by the problem

  • {0, 1, 1, x} OR {1, 0, 0, x} : Sheep and Grass should not be on the same bank without man
  • {0, 1, x, 1} OR {1, 0, x, 0} : Lion and Sheep should not be on the same bank without man

Definition: Traversing between nodes realistically translates to one movement of boat. \({Left Bank} \rightarrow {Right Bank} \rightarrow {Left Bank}\)

e.g. \(\space{A} \rightarrow {K}\space\) basically means: Man and grass on the right, sheep and Lion on the left

Notice: that this traversal can’t be continued further because K = {0, 1, 0, 1} : forbidden state.

To visualise the search algorithm, we can draw a search tree. To dive deeper into the problem, I’ll be making the search tree for finding all possible solutions.

defining constraints in a bool function

Code 
 def isAllowed(node):
    if node[1] == node[2] and node[0] != node[1]:
         return False
    elif node[1] == node[3] and node[0] != node[1]:
         return False
    else:
         return True

DEfining the edges:

now that we have a boolean constraint checker isAllowed, we can formally define valid transitions between states.

A transition corresponds to one boat move:

  • The man must always move.
  • He may take at most one of {sheep, grass, lion} with him.
  • The resulting state must satisfy the constraints.

We generate the edge set by iterating over all allowed states and simulating all physically possible boat moves.

Code 
state_map = {tuple(v): k for k, v in nodes.items()}
edges = []

for label, state in nodes.items():
    if not isAllowed(state):
        continue

    for item in [None, 1, 2, 3]:
        
        if item is not None and state[item] != state[0]:
            continue

        next_state = state.copy()
        next_state[0] = 1 - next_state[0]

        if item is not None:
            next_state[item] = 1 - next_state[item]

        
        edges.append((label, state_map[tuple(next_state)]))

This gives us the complete transition relation of the problem, upto forbidden states

Directed Graph Representation

using the above edge set, we now construct a directed graph where:

  • Nodes represent states
  • Edges represent valid boat crossings

Generic Search Algorithm (Breadth-First Search)

We now apply generic search to find an optimal path from:

  • START = A (initial state {1,1,1,1})
  • TARGET = P (goal state {0,0,0,0})

We explicitly track:

  • LIST: the frontier
  • VISIT: visited nodes

The algorithm prints the updated LIST and VISIT at each step. from collections import deque

Code 
from collections import deque

START = "A"
TARGET = "P"

LIST = deque([START])
VISIT = []
parent = {START: None}

print(f"START = {START}, TARGET = {TARGET}\n")


while LIST:
    
    current = LIST.popleft()
    VISIT.append(current)

    print(f"Expanded: {current}")
    print(f"LIST  : {list(LIST)}")
    print(f"VISIT : {VISIT}\n")

    if current == TARGET:
        print("Target Found!")
        break

    if 'G' in locals():
        for neighbor in G.successors(current):
            if neighbor not in VISIT and neighbor not in LIST:
                parent[neighbor] = current
                LIST.append(neighbor)

Extracting the Solution Path

Once the target is reached, we backtrack using the parent map to get the path.

Code 
path = []
node = TARGET
while node:
    path.append(node)
    node = parent[node]

path.reverse()
print("Solution Path:", " -> ".join(path))
Solution Path: A -> M -> E -> O -> C -> L -> D -> P

Interpretation of the Solution

Each transition corresponds to a single river crossing.

Translated back to reality, the sequence implies:

  1. Man takes sheep across
  2. Man returns alone
  3. Man takes lion across
  4. Man brings sheep back
  5. Man takes grass across
  6. Man returns alone
  7. Man takes sheep across

This is a valid solution under all the given constraints.

Full Search Tree Visualization

Finally, we construct the entire search tree, marking:

  • Forbidden states in red
  • Solution path nodes in green
  • Remaining explored nodes in light blue
Code 
tree_edges = []
visited = set([START])
queue = deque([START])

while queue:
    u = queue.popleft()
    
    if u in G:
        for v in G.successors(u):
            if v not in visited:
                visited.add(v)
                tree_edges.append((u, v))
                queue.append(v)

T = nx.DiGraph(tree_edges)


pos = nx.nx_agraph.graphviz_layout(T)
solution_nodes = set(path)

forbidden_nodes = [k for k, v in nodes.items() if not isAllowed(v)]

node_colors = []
for n in T.nodes():
    if n in solution_nodes:
        node_colors.append("lightgreen")
    elif n in forbidden_nodes:
        node_colors.append("salmon")    
    else:
        node_colors.append("lightblue")
